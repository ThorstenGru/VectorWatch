# ◈ VectorWatch

**Real-time vehicle tracker for Air · Sea · Land** — single HTML file, zero dependencies, deployed on GitHub Pages.

🌐 **Live:** [thorstengru.github.io/VectorWatch](https://thorstengru.github.io/VectorWatch/)

---

## What it does

Three exclusive real-time tracking domains on a dark Leaflet map:

| Domain | Source | Coverage |
|--------|--------|----------|
| ✈ **Air** | ADS-B (adsb.lol · adsb.one · airplanes.live · adsb.fi) | Global, airborne only |
| ⚓ **Sea** | AIS via [aisstream.io](https://aisstream.io) WebSocket | Coastal worldwide |
| 🚂 **Land** | GTFS-RT protobuf per country | 11 countries (SE, NO, FI, DE, NL, BE, FR, CH, DK, IE, AT) |

---

## Features (v2.3.0)

### Map
- **Position trail history** — last 35 positions per entity as a fading polyline
  - Air: altitude-coloured segments (blue = cruise >30 000 ft · green = mid · amber = low · red = approach)
  - Sea/Land: solid line in vehicle-type colour
  - Toggle with **⊙ Trail** button in the filter bar; always shown for the selected vehicle
- **Speed vector arrows** — dashed projection line showing estimated position ahead:
  - 3 min (air) · 10 min (sea) · 5 min (land)
  - Auto-shown at zoom ≥ 9 for air/sea, zoom ≥ 13 for land
- **Squawk emergency pulse** — animated red ring on any aircraft broadcasting an emergency (squawk 7700 / 7600 / 7500 or ADS-B emergency flag)
- **Zoom-adaptive labels** — callsign / vessel name / line number appears below each icon at zoom ≥ 11
- Filter bars per domain with live counts

### Detail panel (click any vehicle)
- **Air:** aircraft type, registration, operator, route (origin → destination), altitude/speed/heading, flight phase, squawk status, Intel DB flag (CIA / USAF / GCHQ etc.), altitude history sparkline, watch-list button
- **Sea:** vessel name, MMSI, IMO, type, nav status, destination with port info, **nearest 4 vessels** (click to jump), watch-list button
- **Land:** line, destination, speed, trip ID, country source badge, watch-list button

### Overlays
- Weather radar (RainViewer live tiles, toggle)
- SIGMET / NOTAM layer (aviation weather hazards)
- Coverage overlay showing which areas have live data vs. demo

### Infrastructure
- **Self-hosted libraries** — Leaflet 1.9.4, ProtobufJS 7, MQTT.js 5.3.5 in `libs/` — zero CDN dependency, works offline once loaded
- **GitHub Actions GTFS-RT pipeline** — runs every 5 min, fetches and decodes protobuf server-side, commits JSON to `data/land/{CC}.json`; frontend prefers this cache (<10 min) over CORS proxies
- **Debug log panel** (toggle with `LOG` button) with timestamped entries

## Architecture

```
index.html          ← entire app (~3 500 lines, no build step)
libs/
  leaflet.css       ← Leaflet 1.9.4 (self-hosted)
  leaflet.js
  protobuf.min.js   ← ProtobufJS 7 (self-hosted)
  mqtt.min.js       ← MQTT.js 5.3.5 (self-hosted)
data/
  land/
    NO.json         ← generated by GitHub Actions every 5 min
    NL.json
    BE.json
    SE.json         ← requires SE_GTFS_KEY secret
    FR.json         ← requires FR_GTFS_KEY secret
.github/
  workflows/
    fetch-gtfs.yml  ← cron: every 5 min
  scripts/
    fetch-gtfs.js   ← Node.js protobuf decoder
    package.json
```

**Deploy = commit to `main`.** GitHub Pages auto-publishes within ~2 minutes.

## Air domain

ADS-B position data from four open APIs, tried in order until one responds:

1. `adsb.lol` (direct)
2. `adsb.one` (direct)
3. `airplanes.live` (direct)
4. `adsb.fi` (needs CORS proxy)

Each API covers a 250 nm radius per tile. The viewport is divided into ≤16 tiles fetched in parallel batches of 4. The working API is cached in memory; if it starts returning 429s the app rotates to the next.

Aircraft are classified into categories (commercial, private, military, government, rescue, intelligence, drone, balloon, glider, UFO) using ICAO emitter category codes and a baked-in Intel DB of known hex addresses.

---

## Sea domain

Live AIS stream from [aisstream.io](https://aisstream.io) WebSocket. The app subscribes to the current map viewport and re-subscribes on pan/zoom. Vessels are rendered as hull-shaped SVG icons rotated to true heading.

**Offshore filter** (`🌊 Offshore only` button): hides inland/river vessels at wide zoom levels using 18 bounding boxes covering European river systems (Rhine, Danube, Thames, Seine etc.).

Demo data (25 vessels worldwide) is shown if the WebSocket fails to connect within 10 seconds.

## Land domain

GTFS-RT vehicle positions decoded from protobuf using ProtobufJS.

### Countries

| Code | Country | Source | Notes |
|------|---------|--------|-------|
| 🇸🇪 SE | Sweden | Samtrafiken / Trafiklab | All 21 regions, updated every 2 s |
| 🇳🇴 NO | Norway | Entur national feed | Free, no key, NLOD 2.0 |
| 🇫🇮 FI | Finland | HSL MQTT live stream | Helsinki region, MQTT over WSS |
| 🇩🇪 DE | Germany | DELFI national feed | All Germany |
| 🇳🇱 NL | Netherlands | OV-API national feed | Free |
| 🇧🇪 BE | Belgium | iRail NMBS/SNCB | Free |
| 🇫🇷 FR | France | Île-de-France Mobilités | Requires API key |
| 🇨🇭 CH | Switzerland | opentransportdata.swiss | Requires API key |
| 🇩🇰 DK | Denmark | Rejseplanen | Requires API key |
| 🇮🇪 IE | Ireland | National Transport Authority | Requires API key |
| 🇦🇹 AT | Austria | — | No open GTFS-RT feed exists |

### CORS proxy fallback chain
Direct → `corsproxy.io` → `api.codetabs.com` → `api.allorigins.win`

The winning route per country is cached in `localStorage` so the next poll skips failed proxies. The **GitHub Actions pipeline** eliminates CORS proxies entirely for the countries it covers by serving pre-decoded JSON from the same origin.

### Finland MQTT
Helsinki region uses the [HSL High-Frequency Positioning](https://digitransit.fi/en/developers/apis/4-realtime-api/vehicle-positions/) MQTT feed at `wss://mqtt.hsl.fi:443/` — topic `/hfp/v2/journey/ongoing/vp/#`. Renders ~2 000 vehicles live, pruned after 30 s of silence.

## GitHub Actions pipeline

`.github/workflows/fetch-gtfs.yml` runs on a 5-minute cron:

1. `actions/checkout@v4` — clones the repo with write token
2. `npm install` in `.github/scripts/` (protobufjs + node-fetch@2)
3. `node .github/scripts/fetch-gtfs.js` — fetches each feed, decodes protobuf, writes `data/land/{CC}.json`
4. `git commit` + `git push` if any JSON changed (tagged `[skip ci]` to avoid loops)

The frontend checks `data/land/{country}.json` on every poll. If the file is less than 10 minutes old and contains vehicles, it's used directly — no CORS proxy needed.

## Key code patterns

Five guards that must never be broken:

```javascript
// 1. Domain guard — every async function that touches air data
if(activeDomain !== 'air') return;

// 2. Fetch deadlock prevention
try { isFetching = true; /* ... */ } finally { isFetching = false; }

// 3. Move timer always cleared on domain switch
clearTimeout(moveTimer); moveTimer = null;

// 4. AbortController on country switch
_landFetchController = new AbortController();

// 5. Snapshot check after every await in land fetch
if(activeDomain !== 'land' || landCountry !== _snap) return;
```

## Known limitations

| Area | Issue |
|------|-------|
| SE | Samtrafiken blocks all CORS proxies (TLS fingerprint) — fixed by GitHub Actions pipeline |
| Air | `corsproxy.io` rate-limits at 429 on wide zoom; max 16 tiles per fetch |
| Sea | No vessel cap — busy ports (Rotterdam, Singapore) may accumulate thousands |
| AT | No open GTFS-RT feed exists for Austria |
| MQTT | HSL covers Helsinki region only, not all Finland |
